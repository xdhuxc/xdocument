### 原生 API 和第三方框架

#### 原生 API
1、使用原生 API 有助于加深对基础架构的理解和增强自己的代码能力。

2、使用原生 API 操作复杂。

#### 第三方框架
1、使用第三方框架可以避免基础组件版本变化带来的 API 变化，第三方框架去屏蔽版本差异，可以保持程序的稳定性。

2、如果第三方框架封装的不够好或者有太多 bug，就会增加程序风险。
